# Soak Test Report Template

This document describes the structure and content of soak test reports generated by the soak runner.

## Report Files

Each soak run produces two output files:
- soak-report-{timestamp}.md - Human-readable markdown report
- soak-report-{timestamp}.json - Machine-readable JSON report with all metrics

## Markdown Report Structure

### Header Section
- **Verdict**: PASS or FAIL status of the soak run
  - PASS: Memory trend is flat AND zero silent failures
  - FAIL: Memory trend is growing OR errors detected

### Test Configuration
- Mode: Dry Run (short, for validation) or Full Soak (long, for production validation)
- Scheduled Duration: Requested duration in seconds
- Actual Duration: Actual runtime in seconds
- Interval: Time between workflow invocations in milliseconds
- Runs: Total number of workflow invocations completed

### Memory Analysis
- Average RSS: Mean resident set size across all samples (MB)
- Max RSS: Peak resident set size (MB)
- Min RSS: Minimum resident set size (MB)
- Memory Trend: FLAT (no growth) or GROWING (potential leak)
- Slope (bytes/tick): Linear regression slope showing memory change per interval
- Slope (MB/hour): Extrapolated memory growth if sustained for one hour
- Fit Quality (R-squared): Goodness of fit for the linear regression (0-1)

### Reliability
- Silent Failures: Count of errors or None (pass)
- Errors: Total error count

### Error Details (if applicable)
For each error documented:
- Tick number where error occurred
- ISO timestamp of the error
- Error message

### Samples Table
Detailed per-tick metrics including RSS (MB) and Handles

## JSON Report Structure

The JSON report contains:

**metadata**:
- type: "soak-run"
- timestamp: ISO 8601 start time
- dryRun: Boolean mode flag
- durationMs: Configured duration
- intervalMs: Configured interval

**results**:
- runCount: Number of workflow invocations
- totalDurationMs: Actual elapsed time
- verdict: "PASS" or "FAIL"
- memoryTrend: Statistical analysis
  - slope: Change in bytes per tick
  - slopePerHour: Extrapolated hourly growth
  - r2: Linear regression goodness of fit
  - flat: Boolean memory stability indicator
- memory: Usage statistics
  - samples: Array of memory snapshots
  - avgMb, maxMb, minMb: RSS statistics
- handles: System handles
  - samples: Per-tick values
  - avg, max: Aggregate statistics
- errors: Array of error objects (empty if clean run)
- errorCount: Total error count
- silentFailures: Boolean (true if no errors)

## Interpretation Guide

### PASS Verdict
Indicates:
- Stable memory (flat linear regression slope)
- No errors detected
- Safe for long-running scheduled deployment

### FAIL Verdict
Indicates one or more of:
- Memory growth detected
- Errors occurred during execution
- Investigation required before deployment

### Memory Slope
- Slope < 100 MB per tick: Considered flat and stable
- Slope > 100 MB per tick: Potential memory leak

### R-squared Values
- R-squared > 0.7: Strong memory trend
- R-squared 0.4-0.7: Moderate trend
- R-squared < 0.4: Weak trend, likely stable
