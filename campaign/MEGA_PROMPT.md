# OPERANT MEGA PROMPT: 15-concurrent, 4-wave, 5-hour, walk-away build campaign
How to use: paste PRD.md, ARCHITECTURE.md, ROADMAP.md, and this document into one Claude Code session (plan mode), type "build this in full" above it, approve, walk away. GitHub CLI is already authenticated. The session ends with a public repo, a tagged v1.0.0 release with a signed installer, a zero-code onboarding experience, a published benchmark, a registry with an in-app gallery, a deployed docs site, and a marketing README with captured screenshots and GIFs.
Concurrency model: up to FIFTEEN lanes run simultaneously in every wave. Four waves, roughly sixty packets total: the wave tables below plus a BACKLOG (section 4b) that keeps concurrency saturated. Whenever a wave slot frees up (a packet merges early or parks), immediately dispatch the highest backlog packet whose dependencies have merged. Slots are never left idle while eligible backlog work exists. Wave 1 saturates concurrency by running foundations alongside everything with no code dependencies (UX foundations, harness, docs scaffold, registry index, launch drafts). The ledger, not the plan, bends under clock pressure.
Design bar carried by every user-facing packet: a person who has never opened a terminal must be able to install, set up a model, teach, run, and schedule a workflow without seeing code or jargon. Developers get power behind an Advanced toggle. PRD section 4.8 and ARCHITECTURE C19 govern; the first-timer path E2E enforces.

## 0. Campaign parameters
- Source of truth: `docs/PRD.md`, `docs/ARCHITECTURE.md`, `docs/ROADMAP.md` (written in Phase 0 from the pasted documents; referenced by path, never restated).
- Wall clock: 5 hours. Phase 0 20 min, Wave 1 70, Wave 2 75, Wave 3 70, Wave 4 plus final gate 65. Checkpoint commit at every wave boundary and every 60 minutes. Behind at a checkpoint: apply the ledger immediately; never compress verification.
- Stack: Rust workspace core, TypeScript SDK, Tauri v2 UI, SQLite, sidecars. Windows tier 1; perception behind the `Perceiver` trait.
- License Apache 2.0, free, no telemetry without opt-in.
- Style, machine-enforced: no em dashes anywhere (CI greps and fails). Conventional commits. ADRs for deviations. UI strings lint against the microcopy glossary so jargon cannot reach default mode.
- Safety, never ledgered: hard invariants (credential fields, payment/delete confirmations) runtime-enforced from the first runnable build.
## 1. Model tier policy (token economy)
Subagents in `.claude/agents/` with explicit `model` frontmatter; every dispatch goes through the tier-matched agent.
| Tier | Model | Packets | Why |
|---|---|---|---|
| Strong | opus | L6A, L6B, L8A, L8B, U4A, X2, X5 | Invariant enforcement, trajectory-to-code, drift repair, the plain-English renderer, the undo journal (a wrong inverse destroys data), and watch-and-suggest (a wrong suggestion erodes the opt-in trust model) |
| Default | sonnet | All implementation and copy packets not listed elsewhere (including X16 oauth-broker) | Well-specified work against frozen contracts |
| Economy | haiku | L10A, L10B, D1A/B/C, C1A, R1A, B1A, S1A, U3B, U5A, M1B, V3, X3, X7, X10, X13, X14, X15, GIF conversion | Mechanical, checkable |
Escalation: a packet failing its bar twice redispatches one tier up, failure log appended, worktree preserved. Never de-escalate. Strong tier failing twice: park, ledger, move on.
Global token rules (encoded in operant-lane): one-screen briefs by path; lanes never read other lanes' source or transcripts; fixtures are the only shared surface; all output through `tail -n 40`, full logs to `scratch/logs/` by path; orchestrator reads RESULT.md only; no unbriefed dependencies without an ADR line.
