Privacy and Security

How Operant keeps your data local and your workflows auditable.

Local-First Architecture

Operant runs entirely on your machine by default. Screen data never leaves your device unless you explicitly configure cloud models with an API key.

Air-Gap Mode

Run Operant completely offline: use a local language model (via Ollama or llama.cpp), no internet connectivity required after initial setup.

Data Security

  • Compiled workflows are plain TypeScript files, inspectable and auditable
  • Credentials stored only in the OS credential vault, never in workflow files
  • Anchor redaction automatically removes sensitive regions from stored screenshots
  • Audit log is cryptographically signed and append-only

Capability Grants

Each workflow declares what it needs: which apps it can control, which paths it can access, what risk classes of actions it can perform. You approve these grants before the workflow runs.

Safety Guarantees

  • No workflow can type into a credential field without explicit approval
  • No workflow can confirm a payment or deletion without manual intervention
  • The kill switch hotkey instantly freezes all input synthesis (under 100 ms)

Transparency

View the plain-English step-by-step execution plan before any workflow runs. Dry-run mode shows what will happen without side effects.
