Your First Workflow

Build and save your first automated workflow in 15 minutes.

What You'll Learn

By the end of this guide, you will have created, saved, and run your first workflow. No coding required. This takes about 15 minutes.

Before You Start

Pick a task you do by hand repeatedly. Some examples:

  • Copy data from a website into a spreadsheet.
  • Fill in a form with the same information each time.
  • Rename files based on their content.
  • Move files from one folder to another.

Step 1: Open Operant and Describe the Task

Click the Operant icon in your system tray to open the command palette. Type what you want done in plain language ("copy the invoice total from the website into the spreadsheet"). You can type it or use your microphone. Then pick which open app the task should run in, so it targets the app you mean and not Operant itself.

Step 2: Watch the Model Work It Out

A model drives the task live on your desktop, figuring out each step as it goes: opening the app, clicking, typing, moving files. You watch each step appear in the flight recorder and turn green as it lands. This first run uses the model; it is the one time it thinks the task through.

Step 3: Save It as a Workflow

When the run finishes, click "Save as Workflow". Operant compiles the run into a numbered list of steps in plain English. You can edit any step, fix mistakes, or add parameters like "which file to copy" or "where to paste it".

Step 4: Run It Again

Click "Run" and watch Operant do it all over again, this time without you touching the mouse or keyboard. This run makes no model calls at all: Operant already knows exactly what to do, so it just replays the saved steps, fast and offline.

Step 5: Run It Whenever You Need It

From now on, one click reruns the task any time. Running it on a schedule with no click at all (every Monday at 9 AM, or when a file arrives in a folder) is on the roadmap: the scheduler is built and tested, but starting a schedule from the app is not wired up yet. The Scheduling guide explains what the engine can already do.

Troubleshooting

If the workflow does not work perfectly the second time (maybe a button moved or a form looked different), Operant will show you what changed and ask if you want to fix it. You can approve the fix in one click and move on.

What Happens Next?

Your workflow is now saved and ready to use whenever you need it. You can run it again with one click, edit it, or share it. The "Scheduling" guide covers the scheduler engine and what running on a schedule will look like once it is wired to the app.
