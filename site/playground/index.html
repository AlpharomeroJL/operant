Watch teach once, replay forever, run in your browser

This page loads the exact same deterministic replay executor that ships in the desktop app (crates/replay), compiled to WebAssembly, and runs it against a fixture invoice app below. No server, no model call, no network request after the page loads: the compiled workflow is a static file, and the wasm module verifies every step, including a real postcondition check, entirely on your machine.

    How this works

    • The workflow was taught once (record, then compile) against this exact fixture page; the compiled workflow is fixtures/compiled_workflow.json, checked in as a static asset.
    • Clicking replay loads pkg/operant_replay.wasm and calls its one exported function, replay_fixture, with the compiled workflow and the fixture page's markup.
    • Inside the wasm module, the same Replayer and browser namespace adapter the desktop app uses dispatch each step against an in-memory copy of the page, including a real assert step against the button's label: rename or remove that button and replay fails, the same way contracts/fixtures/webapp/drift.html does in the test suite.
    • This page mirrors that verified replay into the visible iframe on the left, step by step, so you can watch it happen rather than take the pass/fail summary on faith.
