# Spec: perception

UIA via windows-rs COM interop. Snapshot: walk from the target HWND root with a cached UIA CacheRequest (Name, ControlType, AutomationId, BoundingRectangle, IsEnabled, IsOffscreen, Value and Toggle patterns) in ONE cross-process round trip per subtree; never per-property calls in a loop. Normalize to the schema element: role (ControlType mapped to a fixed enum), name, value, bounds in physical pixels PLUS the monitor's DPI scale factor so logical coordinates reconstruct on any display arrangement. Selector chain per element, most stable first: (1) AutomationId if nonempty and unique in scope, (2) role plus name path from the window root, (3) role plus ordinal among siblings. Store all three; replay tries them in order. Snapshot digest: BLAKE3 over the normalized element list minus bounds (layout changes must not change the digest; content changes must). Diff: keyed on selector chains, emits added/removed/renamed/value-changed. wait_until_changed(scope, timeout): subscribe to UIA structure and property events where available, poll-diff at 100 ms as fallback. Virtualized lists (UIA VirtualizedItem pattern): realize on demand, cap realization at 500 items per snapshot with a truncation flag in the snapshot. Known limits documented and tested: elevated (admin) windows are invisible without elevation, return a typed PerceptionDenied error, never an empty tree; secure desktop (UAC) likewise. Multi-monitor: all coordinates carry the monitor handle. Target: Notepad full snapshot under 200 ms, Explorer under 400 ms.
