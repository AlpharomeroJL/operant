# Spec: action

Input via SendInput with these rules: focus the target window (SetForegroundWindow with the attach-thread-input workaround) and verify focus by re-reading the focused element BEFORE any keystroke; typing uses KEYEVENTF_UNICODE so keyboard layout never corrupts text; modifier keys always paired press/release with a panic-time release-all sweep (kill switch depends on it). Click: resolve selector chain to fresh bounds at execution time (never cached coordinates), click the clickable point from UIA if the pattern provides one, else bounds center; post-click, verify by re-snapshot of the target scope; on verification failure retry per the Action IR retry field (default 2 retries, 250 ms backoff), then fail typed. Scroll: prefer the UIA Scroll pattern; wheel fallback. Pacing field: instant (default for replay), human (40 to 120 ms jittered inter-key, 150 to 400 ms pre-click hover). Adapter framework: an adapter registers a namespace, a JSON schema per verb, a risk_class per verb, and an idempotency hint; the executor validates params against the schema before dispatch. Shell adapter: PowerShell with a 60 s default timeout, stdout/stderr captured to artifacts, exit code in the outcome. Filesystem adapter verbs: read, write, copy, move, delete (delete is destructive risk class and routes through recycle bin so the undo journal holds). Office COM: Excel (open workbook, read range, write range, save) and Word (open, get text, replace text, save), each releasing COM objects deterministically. Email: IMAP fetch/search and SMTP send (send is irreversible, labeled). OCR/PDF: extract text plus word bounding boxes from images and PDFs, on device.
