# Spec: scheduler

Trigger types and specs: cron (5-field, local time, DST-safe via next-occurrence computation), file-watch (directory plus glob, debounced 2 s, event carries the matched path as a workflow input), window-appears (process name plus title regex, polled at 2 s via the perception layer), email-arrives (IMAP IDLE where supported, poll fallback, subject/from filters, message id as input). Queue: serialized by default; a run declares its capability scope and two runs may parallelize only if scopes are disjoint (app sets and directory subtrees do not intersect and neither has network). Missed triggers while the machine slept: cron fires once if within a 15-minute grace, file events are re-scanned on wake, everything else drops with a log line. Unattended enforcement: the scheduler refuses to enqueue any run whose mode is not replay; the refusal is typed and tested.
