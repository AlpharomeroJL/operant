# Spec: registry

Manifest: name, version (semver), publisher, pubkey fingerprint, description, plain-English step summary (from the renderer, embedded so the gallery can show it pre-install), inputs schema, capability requirements, min Operant version, Ed25519 signature over the canonical JSON. Index repo layout: /index.json (name -> versions -> manifest path), /manifests/<name>/<version>.json, /keys/<publisher>.pub. Install flow: fetch manifest, verify signature against a key the user has pinned (first install of a publisher shows the fingerprint and asks once), render the embedded step summary and grants in plain language, require approval, then fetch the DSL file (hash-pinned by the manifest), store, grant, done; any hash or signature mismatch aborts with a typed error. Unsigned or unpinned: install allowed but the workflow is flagged and executes in dry-run only until the user explicitly promotes it after reading the steps. Publish: `operant publish` signs with the local publisher key, emits a branch plus PR body against the index repo.
