# Spec: bench

Suite: the fixture Notepad task, the fixture web task, the drift fixture (post-repair), plus three cookbook workflows. Modes: replay (compiled), re-inference-mock (the mock planner re-plans every step, simulating agent-every-time cost with recorded latencies), and re-inference-real behind an env flag when a real backend is configured. Metrics per task per mode: success boolean over 5 repetitions, p50 and p95 per-step latency, total wall time, model calls, tokens (0 for replay by construction). Output: BENCHMARKS.md with a headline table and a methods section stating exactly what was measured and that re-inference-mock uses recorded latencies (honesty note in the file itself). CI threshold: replay success 5/5 on unchanged fixtures and p50 step under 150 ms, else fail.
