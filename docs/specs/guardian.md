# Spec: guardian

Kill switch: a low-level keyboard hook (WH_KEYBOARD_LL) registered by a tiny dedicated thread in the core process, watching the panic chord (default Ctrl+Alt+Shift+Space, configurable); on trigger: set a process-wide atomic that the action layer checks before EVERY SendInput batch, release all held modifiers, cancel in-flight adapter calls cooperatively with a 500 ms grace then abandon, mark all active runs halted-by-human in the recorder, tray red. The atomic check path is lock-free; latency test asserts trigger-to-frozen under 100 ms while a run is mid-typing. Resume is per-run and explicit. Undo journal: at execution time, write-class actions append their inverse BEFORE performing the action (journal-ahead): file create -> delete; file delete -> recycle-bin restore token; file move/rename -> reverse move; file overwrite -> pre-image copied to the blob store, inverse restores it; clipboard write -> prior clipboard restored. Undo replays inverses newest-first inside a dry-run preview first, then executes on confirm, narrating each restoration. Actions with no inverse (email send, web form submit, shell with side effects) are tagged irreversible at the adapter schema level; the renderer badges them and "Undo last run" lists them as "cannot be undone" rather than pretending. Redaction: the capture path calls redact(snapshot, image) before any blob write; regions from IsPassword elements and from the safety classifier's credential-dialog match are filled solid; if redaction errors, the write is refused (fail closed).
