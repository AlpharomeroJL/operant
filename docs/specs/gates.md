# Spec: gates

Gate expression language: a small JSON predicate AST (no strings-of-code), operators exists, equals, matches (anchored regex), count, sum, withinTolerance, and selectors that address the perception snapshot, filesystem (path, hash, size), and adapter results. Precondition gates run before step 1 and before any step marked risky; postconditions per compiler pass 4; safety gates are runtime-owned: a deny-list evaluated on EVERY proposed action in explore and every step in replay: target element is a password/credential field (UIA IsPassword or role match), foreground dialog classified as payment or deletion confirmation (classifier: role tree shape plus title/button lexicon, lexicon in a data file), or an unexpected window class appearing mid-run. Safety gate hit: freeze the run (not the machine), emit Escalation with a plain-language sentence, require an explicit human approval event to proceed; approvals are recorded in the audit chain with the exact proposed action. Grants: scopes are app (by executable path or package id), directory subtree, network (boolean at v1), and risk class ceilings; checked at execution time by the action layer, not at plan time; check failure is a typed refusal the planner can react to in explore and a halt in replay. Audit: every event (action executed, gate result, approval, grant change, workflow install, drift merge) appended as JSONL with prev_hash chaining (BLAKE3); verify command walks the chain; PDF export renders human-readable pages with the chain head hash on every page.
