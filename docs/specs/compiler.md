# Spec: compiler

Input: a trajectory (steps table rows for one run). Pass 1 normalize: drop steps whose outcome is retry-superseded; where a HumanCorrection event exists, keep the corrected branch and drop the corrected-away steps. Pass 2 parameterize: a literal becomes a typed input when (a) it varies across runs of the same goal, or (b) it matches date, currency-amount, file-path, email, or URL shapes; inferred inputs get name, type, default, and appear in the manifest inputs schema; everything else stays constant. Pass 3 selectorize: score each stored selector (AutomationId 100, name+role path 60 minus 5 per path level, ordinal 20); attach the vision anchor as last resort with tolerance 0.85 template match; emit all with order. Pass 4 waits and asserts: after every action that changed the snapshot digest, insert wait_until_changed on the tightest scope that changed; after the final step and after any step the recorder marked outcome-bearing (file created, row added), insert a postcondition assert from the observed state. Pass 5 emit: a TypeScript file over the SDK (readable: one statement per step, comments carrying the plain-English step text) plus the manifest JSON (name, version, inputs schema, capability requirements computed from the union of step needs, gate bindings, signature block). Replay executor: interprets the same DSL through the action layer with pacing instant, model calls structurally impossible (the executor links against a backend-free crate; enforced by the crate graph, not a runtime flag), network denied by test. Composition (X6): a callWorkflow step; the executor loads the callee manifest, intersects grants, and refuses at load if the intersection cannot satisfy the callee's requirements, with a plain-language explanation.
