# Spec: drift

Detection: during replay, a step fails drift-eligible when all selectors miss OR the anchor match falls below tolerance, while the precondition gate for the step still holds (wrong-state failures are NOT drift and halt normally). Repair: capture the current snapshot, re-ground the single step target with the grounder against the step's recorded plain-English intent, produce a candidate selector chain plus anchor. Patch: a diff file (old selectors/anchor -> new, with before/after screenshots) written to the workflow's pending-patches dir; the run halts as "needs your OK" not "failed". Approval: UI card or `operant approve <workflow> <patch>` shows the plain-language offer ("The Save button moved. Update the workflow?") with the two screenshots; approval bumps the workflow version, appends to workflow_versions with the diff path, and re-runs the failed step onward. Rejection archives the patch. Never: auto-merge, multi-step repair in one patch, or repair of a step whose risk class is destructive (those always halt to a human).
