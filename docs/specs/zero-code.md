# Spec: zero-code

Wizard screens, in order: (1) welcome, one sentence, one Continue; (2) "How should Operant think?" with four cards: Sign in with ChatGPT, Sign in with Claude (both: "Use the AI plan you already pay for. No keys to copy."), "Download a free brain" (local; shows required disk and works-on-this-PC check from the VRAM probe; progress bar with pause/resume), "I have an API key" (paste field, provider auto-detected from key shape where possible, dropdown otherwise), plus a quiet fifth link "Just show me a demo"; (3) mic check IF the user opts into voice, with an audible sample and a visible level meter (media-presence check in CI); (4) guided first task: a suggested goal against the fixture web app, narrated explore, ending on one button labeled "Save as workflow"; (5) schedule it: "Want this to run by itself?" with plain schedule choices. Renderer rules: every Action IR kind has a sentence template (click -> 'Click "{name}"', type -> 'Type {value} into "{field}"', adapter_call file.move -> 'Move {src} to {dest}', wait -> 'Wait for the screen to update', assert -> 'Check that {condition, rendered}'); parameters render as inline form chips; unknown-kind fallback is FORBIDDEN in default mode (property test: rendering is total). Error catalog entry format: id, what (one sentence, past tense), why (one sentence), action (one imperative sentence), fix_command (optional, powers the one-click fix), learn_more (docs anchor). Doctor checks: model reachable (per configured backend), OAuth token valid, disk free above threshold, accessibility permission granted, updater reachable, audio devices present, VRAM headroom for configured sidecars.
